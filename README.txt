Obstacle.h - מחלקה שנועדה לייצוג כל סוגי האובייקטים במשחק שניתן להרוס ושהשחקן יכול לבוא איתם במגע ולקבל תגובה מהם.
המחלקה מנהלת את עצמה וכל אובייקט גרפי עוקב אחרי האובייקט הפיזיקלי, והאובייקט הפיזיקלי מנוהל ע"י התזוזות במשחק עצמו וע"י פגיעות מהשחקן שמנוהלות ע"י השחקן.

Circular Obstacle.h - מחלקה שנועדה לייצוג כל סוגי האובייקטים העגולים במשחק, ההבדלים בינה לבין Obstacle הוא גם במימוש הפונקציות הגרפיות והפונקציות הפיזיקליות הרי שהאובייקטים עגולים ולכן רצינו התנהגות שונה מאשר האובייקטים שאינם עגולים במשחק.

GuideLine.h - מחלקה שנועדה לייצג את הכוונת של השחקן כאשר הוא מותח את הציפור, בעת מתיחה הכוונת תתנהג ותדמה את זריקת הציפור ולאיפה היא תעוף אילו השחקן יעזוב את העכבר וישלח את הציפור לדרכה, המחלקה משתמשת באובייקטים פיזיקליים כדי לדמות את זריקות הציפור בעזרת אובייקט פיזיקלי של Box2d שנקרא sensor ולכן איננו משתתף ב collisions במשחק ולכן איננו מפריע ופוגע באובייקטים.

LevelManager.h - מחלקה שנועדה לקרוא את השלבים מקובץ הטקסט שמייצג את השלבים וליצור את האובייקטים המתאימים ולהכניסם לוקטורים שיתנו לו ע"י התוכנית. התוכנית ממקמת את האובייקטים ע"פ כללים שנקבעו על ידינו בקובץ הטקסט שלנו ועבור סימנים מוסכמים מראש.
המיקומים נקבעים בהתאמה ובאופן יחסי לגודל המסך שלנו.
